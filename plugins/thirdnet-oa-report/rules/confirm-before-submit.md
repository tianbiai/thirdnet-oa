# 写操作必须用户确认

任何对 OA 系统的写操作（日报提交、催办发送），在调用 MCP 接口之前，必须完成以下两步：

## 1. 展示预览

将即将提交的完整内容以可读格式展示给用户，包括：
- 目标机构名称和 ID
- 日期
- 所有字段内容（按 field_name 展示）
- 催办场景下展示催办对象名单

## 2. 等待用户确认

使用 AskUserQuestion 工具明确询问用户是否确认提交。必须等用户回答"确认"/"提交"/"是"等肯定答复后，才可继续调用接口。

如果用户选择修改，则更新内容后重新展示预览并再次确认。

## 为什么这条规则很重要

用户可能对 AI 生成的内容不满意，或需要补充/修改某些条目。跳过确认直接提交会导致错误的日报被记录到 OA 系统，而 OA 没有覆盖/撤回接口，一旦提交就无法修改。这是不可逆操作，必须由用户把关。
